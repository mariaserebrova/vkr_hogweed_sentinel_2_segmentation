# Поиск крупных очагов борщевика Сосновского по снимкам Sentinel-2

Репозиторий содержит программную реализацию итогового эксперимента для выпускной квалификационной работы, посвящённой автоматическому выявлению крупных очагов борщевика Сосновского на многоспектральных спутниковых снимках Sentinel-2.

Задача рассматривается как бинарная семантическая сегментация. На вход модели подаются многоканальные фрагменты спутникового растра, на выходе строится карта вероятности принадлежности каждого пикселя классу «борщевик Сосновского».

В итоговой конфигурации использовалась модель U-Net++ с энкодером ResNet34 и десятиканальным входом:

`B2, B3, B4, B5, B6, B7, B8, B8A, B9, B11`.

---

## О репозитории

Итоговый эксперимент изначально выполнялся в Jupyter Notebook.  
В папке `notebooks/` сохранена его исполненная версия со всеми output’ами: графиками обучения, таблицами метрик, полноразмерными картами, результатами объектной оценки и итоговыми визуализациями.

Отдельно код был вынесен в структурированный проект. Это позволяет разделить основные этапы пайплайна:

- загрузку подготовленных многоканальных растров и маски;
- формирование обучающей, валидационной и тестовой выборок;
- обучение модели;
- оценку качества;
- построение полноразмерного прогноза;
- расчёт объектных и буферизованных метрик.

Таким образом, ноутбук сохраняет полный ход финального эксперимента и его результаты, а проектная структура делает программную часть более удобной для просмотра и повторного запуска отдельных этапов.

## Содержимое проекта

### `config.py`

В этом файле собраны основные параметры эксперимента:

- пути к данным;
- даты съёмки;
- используемые спектральные каналы;
- параметры формирования окон;
- настройки обучения;
- параметры полноразмерного прогноза;
- пороги и настройки расчёта метрик.

### `src/`

Основные модули проекта:

- `raster_io.py` — чтение растров и маски;
- `windows.py` — формирование окон и пространственное разбиение;
- `dataset.py` — датасет и загрузчики данных;
- `model.py` — архитектура модели;
- `training.py` — обучение и валидация;
- `metrics.py` — расчёт пиксельных, объектных и буферизованных метрик;
- `inference.py` — полноразмерный прогноз;
- `visualization.py` — построение графиков, карт и итоговых рисунков;
- `pipeline.py` — сборка основных этапов эксперимента.

### `scripts/`

Скрипты для запуска отдельных частей пайплайна:

- `train_model.py` — обучение модели;
- `evaluate_model.py` — оценка на validation/test;
- `full_raster_report.py` — полноразмерный прогноз и итоговые метрики;
- `make_rgb_dates_figure.py` — построение RGB-композитов выбранных сцен.

### `notebooks/`

В папке находится исполненный ноутбук:

```text
hogweed_unetpp_stratified_large_focus_tversky.ipynb
```

Он сохранён вместе с результатами выполнения ячеек и содержит полный финальный эксперимент: подготовку входных выборок, обучение модели, подбор порога, оценку на тестовой выборке, расчёт метрик для полной области исследования и итоговые визуализации.

## Данные

Проект работает не с исходными файлами Sentinel-2, а с заранее подготовленными многоканальными растрами, приведёнными к единой геометрии и обрезанными по области исследования.

Полные растры в репозиторий не включены из-за объёма.  
Пути к данным задаются в `config.py`.

По умолчанию используется следующая структура:

```text
prepared_10m/
├── 0512/
│   └── final/
│       └── *_final_clip_12ch.tif
├── 0718/
│   └── final/
│       └── *_final_clip_12ch.tif
├── 0904/
│   └── final/
│       └── *_final_clip_12ch.tif
├── 0907/
│   └── final/
│       └── *_final_clip_12ch.tif
├── 0909/
│   └── final/
│       └── *_final_clip_12ch.tif
└── _shared/
    └── mask_final_clip.tif
```

При другой структуре каталогов достаточно изменить соответствующие пути в конфигурационном файле.

## Запуск

### Обучение модели

```bash
python scripts/train_model.py
```

Лучший чекпоинт сохраняется в каталог, указанный в `config.py`.

### Оценка на validation/test

```bash
python scripts/evaluate_model.py
```

Скрипт использует сохранённый чекпоинт модели и рассчитывает основные метрики качества, а также выполняет подбор порога бинаризации.

### Полноразмерный прогноз

```bash
python scripts/full_raster_report.py
```

Сценарий строит прогноз для полной области исследования, рассчитывает итоговые пиксельные, объектные и буферизованные метрики, а также сохраняет таблицы и визуализации.

### RGB-композиты выбранных дат

```bash
python scripts/make_rgb_dates_figure.py
```

## Итоговая конфигурация

- архитектура — `U-Net++`;
- энкодер — `ResNet34`;
- вход — 10 каналов Sentinel-2;
- функция потерь — `0.4 BCE + 0.6 Tversky`;
- параметры Tversky — `α = 0.30`, `β = 0.70`;
- вес положительного класса в BCE — `4`;
- итоговый порог бинаризации полноразмерной карты — `0.75`.

## Назначение проекта

Проект ориентирован на предварительный автоматизированный мониторинг крупных очагов борщевика Сосновского по спутниковым данным.

Полученные карты не предназначены для юридически точного картографирования границ ареала и не заменяют экспертную проверку. Их основная задача — выделять зоны повышенного внимания, которые требуют дальнейшего анализа и, при необходимости, наземного обследования.
